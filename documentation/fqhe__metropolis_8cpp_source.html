634 //! The sphere. To move the point in an unbiased way, we must make the move in
635 //! the reference frame of the north pole:
636 //!
637 //! - Pick some point to move, with co-ordinates theta (denote it by "t_old")
638 //! and phi (denote it by "p_old") on a unit sphere.
639 //!
640 //! - We want to pick a random distance and direction to move the point in.
641 //! This is done by making a random 3D Cartesian vector dx with components
642 //! between -a and +a, where "a" is the maximum move distance. Then I make
643 //! dx+(0,0,1) and normalize it. This vector is now a unit vector pointing
644 //! to a point on the surface of the sphere almost at the north pole.
645 //! I think that there will be a uniform distribution of displacements of
646 //! that vector from (0,0,1).
647 //!
648 //! - Next we rotate this vector to the vicinity of the point that we want to
649 //! move (I will explain why this has to be done shortly). From the co-ordinates,
650 //! {t_old,p_old} we make a rotation matrix that will take the vector (0,0,1)
651 //! and map it to the direction {t_old,p_old}. The formula I use to get this
652 //! is given here: http://en.wikipedia.org/wiki/Rotation_matrix (scroll down
653 //! to where it says: "Rotation matrix from axis and angle").
654 //! Let's call this R(t_old,p_old). I then use R(t_old,p_old) to rotate the
655 //! random unit vector (dx+(0,0,1) normalized) to the location of the point
656 //! we wanted to move. This gives some new Cartesian vector (x,y,z) and then
657 //! from that I can get:
658 //!
659 //! p_new = arctan(y,x)
660 //! t_new = arccos(z)
661 //!
662 //! which are very close to the original p_old and t_old. The point is that
663 //! the value of {x,y,z} that one would get for the same dx would depend on
664 //! R(t_old,p_old). So you could not just take the original point {t_old,p_old} ,
665 //! add dx to it and normalize the result to unit length.
666 //!
667 //////////////////////////////////////////////////////////////////////////////////

1070  double energy=(double)8*n/(3*PI*wfData->discRadius);

1072

1074  {

1084  }

1087  {

1090  }
1091  }
1092
